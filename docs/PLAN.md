# PLAN.md — Phase별 개발 목표

## 개요

각 Phase는 **실제로 브라우저에서 실행되고 플레이 가능한 상태**를 목표로 한다.
고객님은 Phase 완료 시마다 `npm run dev`로 직접 플레이하며 피드백을 주시면 된다.
각 Phase 배포 전, Computer Use(알파 테스터)가 브라우저를 직접 조작해 기본 동작을 사전 검증한다.

---

## Phase 1 — 게임 화면과 플레이어 이동

> **목표**: 게임판이 열리고 캐릭터를 좌우로 움직일 수 있는 상태

### 구현 내용
- Canvas 기반 게임 화면 구성 (480×640 고정 해상도)
- 플레이어 캐릭터 렌더링 (사각형 또는 단순 스프라이트)
- 좌우 이동 (← →), 화면 경계에서 멈춤

### 알파 테스터 검증 항목
- [ ] 브라우저에서 검은 배경 위에 게임 화면이 표시되는가
- [ ] 방향키 ← →로 캐릭터가 좌우로 움직이는가
- [ ] 화면 끝에서 캐릭터가 벗어나지 않는가

### 고객님 확인 포인트
- 이동 속도가 손에 맞는 느낌인지 (너무 빠르거나 느리지 않은지)
- 화면 비율이 플레이하기 적당한 크기인지

---

## Phase 2 — 하푼 발사

> **목표**: 버튼을 누르면 하푼이 위로 날아가는 상태

### 구현 내용
- Space / Z 키로 하푼 발사 (위 방향 고정)
- 하푼이 천장에 닿으면 소멸
- 단발 제한 — 이전 하푼이 있을 때는 재발사 불가

### 알파 테스터 검증 항목
- [ ] Space/Z 키로 하푼이 위쪽으로 발사되는가
- [ ] 하푼이 천장에 닿으면 사라지는가
- [ ] 하푼이 공중에 있는 동안 재발사가 막히는가

### 고객님 확인 포인트
- 하푼 발사 속도가 너무 빠르거나 느리지 않은지
- 단발 제한이 있을 때 조작이 답답하게 느껴지지는 않는지

---

## Phase 3 — 풍선 물리 운동

> **목표**: 풍선 1개가 바닥을 통통 튀며 화면을 돌아다니는 상태

### 구현 내용
- 대형 풍선 1개 배치
- 중력 + 초기 속도로 포물선 운동 구현
- 바닥 반사 (튀어오름), 좌우 벽 반사

### 알파 테스터 검증 항목
- [ ] 풍선이 바닥에서 튀며 이동하는가
- [ ] 좌우 벽에 닿으면 방향이 반전되는가
- [ ] 튀어오르는 높이가 자연스럽게 포물선을 그리는가

### 고객님 확인 포인트
- 풍선이 통통 튀는 느낌이 원작과 비슷하게 느껴지는지
- 이동 속도가 너무 빠르거나 느리지 않은지

---

## Phase 4 — 충돌 감지

> **목표**: 하푼이 풍선에 맞고, 플레이어가 풍선에 닿으면 반응하는 상태

### 구현 내용
- 하푼 ↔ 풍선 충돌 감지 → 풍선 및 하푼 제거
- 플레이어 ↔ 풍선 충돌 감지 → 캐릭터 깜빡임 (이 Phase에서는 사망 처리 없이 시각 반응만)

### 알파 테스터 검증 항목
- [ ] 하푼이 풍선에 닿으면 두 오브젝트가 모두 사라지는가
- [ ] 플레이어가 풍선에 닿으면 캐릭터가 깜빡이는가
- [ ] 충돌 판정이 눈에 보이는 크기와 어긋나지 않는가

### 고객님 확인 포인트
- 충돌 판정이 느슨하거나 억울하게 맞는 느낌이 드는지
- 하푼이 풍선을 맞혔을 때 반응이 명확하게 느껴지는지

---

## Phase 5 — 풍선 분열 시스템

> **목표**: 풍선을 맞추면 작은 풍선 2개로 쪼개지고 최소 크기는 완전히 사라지는 상태

### 구현 내용
- 풍선 4단계 분열 메커니즘 구현
  ```
  대 → 중×2 → 소×4 → 극소×8 → 제거
  ```
- 분열 시 좌우로 퍼지는 속도 적용
- 크기가 작을수록 이동 속도 증가, 튀어오르는 높이 감소

### 알파 테스터 검증 항목
- [ ] 대형 풍선을 맞추면 중형 2개가 생성되는가
- [ ] 4단계까지 분열이 이어지는가
- [ ] 극소 풍선을 맞추면 완전히 사라지는가
- [ ] 분열된 풍선이 좌우로 퍼지는가

### 고객님 확인 포인트
- 풍선이 분열될 때 좌우로 자연스럽게 퍼지는지, 한 자리에 겹쳐 쌓이지 않는지
- 크기가 작아질수록 빨라지는 속도 변화가 체감되는지

---

## Phase 6 — 생명 시스템과 게임 오버

> **목표**: 풍선에 닿으면 죽고, 잔기가 다 떨어지면 게임 오버가 되는 상태

### 구현 내용
- 잔기 3개 시작
- 풍선 충돌 시 즉사, 잔기 1 감소 후 현재 스테이지 재시작
- 잔기 0에서 사망 시 게임 오버 화면 표시
- 게임 오버 화면에서 재시작 버튼 (처음부터)

### 알파 테스터 검증 항목
- [ ] 풍선에 닿으면 잔기가 1 감소하고 재시작되는가
- [ ] 잔기가 0에서 죽으면 게임 오버 화면이 나오는가
- [ ] 게임 오버 후 재시작하면 잔기 3개로 처음부터 시작되는가

### 고객님 확인 포인트
- 사망 후 재시작까지의 흐름이 자연스럽게 이어지는지
- 게임 오버 화면이 명확하게 전달되는지

---

## Phase 7 — HUD와 타이머

> **목표**: 점수·잔기·타이머가 화면에 표시되고 시간이 다 되면 사망하는 상태

### 구현 내용
- 화면 상단 HUD: 현재 점수, 잔기, 남은 시간 표시
- 제한 시간 60초 카운트다운
- 타이머 0 도달 시 즉사 처리
- 풍선 파괴 시 크기별 점수 획득 (극소 → 소 → 중 → 대 순으로 고득점)

### 알파 테스터 검증 항목
- [ ] 점수·잔기·타이머가 화면에 표시되는가
- [ ] 풍선을 터뜨리면 점수가 올라가는가
- [ ] 타이머가 60초부터 카운트다운되는가
- [ ] 타이머가 0이 되면 사망 처리가 되는가

### 고객님 확인 포인트
- HUD 위치와 크기가 게임 화면을 방해하지 않는지
- 타이머 60초가 플레이 중 충분하게 느껴지는지 (너무 여유롭거나 촉박하지 않은지)

---

## Phase 8 — 스테이지 클리어와 진행

> **목표**: 모든 풍선을 제거하면 클리어되고 다음 스테이지로 넘어가는 상태

### 구현 내용
- 모든 풍선 제거 시 스테이지 클리어 판정
- 클리어 메시지 및 짧은 대기 후 다음 스테이지 전환
- Mission 1 Stage 1 → 2 → 3 순서 진행
- Stage별 풍선 배치 정의 (수량, 크기, 시작 위치)
- Stage 3 클리어 시 Mission 클리어 화면

### 알파 테스터 검증 항목
- [ ] 풍선을 전부 없애면 클리어 메시지가 표시되는가
- [ ] 잠시 후 다음 스테이지로 자동 전환되는가
- [ ] Stage 1 → 2 → 3 순서대로 진행되는가
- [ ] Stage가 올라갈수록 풍선 수가 늘어나는가
- [ ] Stage 3 클리어 시 Mission 완료 화면이 나오는가

### 고객님 확인 포인트
- Stage 1~3의 난이도 차이가 자연스럽게 느껴지는지
- 클리어 후 다음 스테이지로 넘어가는 템포가 적당한지

---

## Phase 9 — 무기 아이템

> **목표**: 풍선을 터뜨리면 무기 아이템이 떨어지고, 주우면 발사 방식이 바뀌는 상태

### 구현 내용
- 풍선 파괴 시 확률적으로 무기 아이템 드롭
- 아이템이 바닥으로 떨어지며 일정 시간 후 소멸
- 무기 3종 구현 및 적용
  - 더블 와이어: 좌우 2발 동시 발사
  - 파워 와이어: 발사 후 천장에 일정 시간 고정
  - 벌칸 미사일: 짧은 간격으로 연속 속사
- 새 무기 획득 시 기존 무기 대체
- 사망 시 기본 하푼으로 초기화

### 알파 테스터 검증 항목
- [ ] 풍선 파괴 시 아이템이 드롭되는가
- [ ] 더블 와이어: 2발이 동시에 발사되는가
- [ ] 파워 와이어: 천장에 고정 후 일정 시간 뒤 사라지는가
- [ ] 벌칸 미사일: 연속으로 빠르게 발사되는가
- [ ] 사망 후 기본 하푼으로 돌아오는가

### 고객님 확인 포인트
- 각 무기의 특성이 플레이 스타일에 영향을 주는 느낌인지
- 아이템 드롭 빈도가 적절한지 (너무 자주 뜨거나 거의 안 뜨거나 하지 않는지)

---

## Phase 10 — 파워업 아이템

> **목표**: 배리어·시계 등 파워업 아이템을 주워서 전략적으로 활용할 수 있는 상태

### 구현 내용
- 파워업 6종 구현 및 적용
  - 배리어: 풍선 충돌 1회 방어 후 소멸
  - 시계: 모든 풍선 일시 정지 (타이머는 계속 진행)
  - 모래시계: 모든 풍선 이동 속도 감소 (일정 시간)
  - 다이나마이트: 화면 내 모든 풍선을 즉시 최소 크기로 분열
  - 1UP: 잔기 1 추가
  - 음식 아이템: 수천 점 보너스 점수

### 알파 테스터 검증 항목
- [ ] 배리어 착용 중 풍선에 닿으면 죽지 않고 배리어만 소멸하는가
- [ ] 시계 획득 시 풍선이 멈추는가 (타이머는 계속 가는가)
- [ ] 모래시계 획득 시 풍선이 느려지는가
- [ ] 다이나마이트 획득 시 모든 풍선이 최소 크기로 한꺼번에 분열되는가
- [ ] 1UP 획득 시 잔기가 1 증가하는가

### 고객님 확인 포인트
- 각 아이템의 효과가 획득 즉시 직관적으로 느껴지는지
- 다이나마이트가 위험 아이템으로 체감되는지 (최소 풍선이 갑자기 가득 차는 긴장감)
- 시계와 모래시계의 차이가 명확하게 느껴지는지

## Phase 11 — 메인 화면과 전체 흐름 연결

> **목표**: 메인 화면 → 게임 → 클리어/게임오버 → 메인으로 이어지는 완전한 게임 흐름

### 구현 내용
- 메인 화면: 타이틀 로고, 1 PLAYER 메뉴, HI-SCORE 표시
- 화면 전환 흐름 완성
  ```
  메인 화면 → Stage 1 → Stage 2 → Stage 3 → Mission 클리어 → 메인
                                ↓
                            게임 오버 → 메인
  ```
- HI-SCORE 세션 내 저장 및 메인 화면 반영
- UI 폴리싱: 폰트, 색상, 레이아웃 정리

### 알파 테스터 검증 항목
- [ ] 메인 화면에서 1 PLAYER 선택 시 Stage 1이 시작되는가
- [ ] 게임 오버 후 메인 화면으로 복귀하는가
- [ ] Mission 클리어 후 메인 화면으로 복귀하는가
- [ ] 세션 내 최고 점수가 메인 화면 HI-SCORE에 반영되는가

### 고객님 확인 포인트
- 처음 게임을 켰을 때 첫인상이 팡 게임답게 느껴지는지
- 모든 화면 간 전환이 자연스럽고 막히는 부분이 없는지
- 전체적으로 "완성된 하나의 게임"처럼 느껴지는지
